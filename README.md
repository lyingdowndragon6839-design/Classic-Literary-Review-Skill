# Classic-Literary-Review-Skill

## 用途

这个项目用来沉淀一套中文名著读后感、文学长评、读书随笔的写作 Skill。它服务的不是课堂摘要，也不是资料汇编，而是让写作者围绕一本书形成稳定判断：从人物、结构、历史、心理、叙事节奏里找到问题，再把问题写成能读下去的长文。

## 不适合写什么

不适合写纯剧情复述、百科式作者生平、考试标准答案、学术论文脚注体系、营销号爽文、读书卡片堆叠，也不适合在没有读过作品细节时伪装成“深度书评”。

## 为什么不以“降 AI 率”为目标

“降 AI 率”容易把写作引向表面替换：换几个词、加几句口语、故意制造不规则表达。这样的目标会让文章只关心检测器，而不关心作品、判断和读者。这个 Skill 关注的是文本细节、思考路径和声音稳定；当这些东西成立时，文章自然会远离模板腔。

## 为什么以“稳定写作者声音”为目标

文学长评最怕每一段都像不同人写的：一会儿学院论文，一会儿短视频解说，一会儿作文结尾。稳定声音意味着写作者有固定的观察习惯、判断尺度、句子节奏和节制感。它不保证每个观点都温和，但要求每个锋利判断都有文本细节支撑。

## 模块边界

`modules/` 保存可单独迭代的写作能力；`examples/` 保存作品级示范；`workflow.md` 保存从读书笔记到成文的流程；`skill.md` 保存最终可复制使用的整合版规则。
