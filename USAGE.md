# 读后感写作使用指南

这个文件给真正要写读后感的人看。

你不需要理解所有内部模块。你只需要把自己的阅读情况说清楚，让 Agent 先帮你搜索、整理、选角度，再写文章、改文章。

## 最推荐的一句话使用方式

```text
我想写《悲惨世界》的读后感，字数 1500 字，风格要自然，像一个认真读完书的人写的。请先联网搜索高质量评论，帮我提炼 3 个可写角度，再选一个最有力量的主判断，搭结构，写初稿，最后帮我降低 AI 味。
```

这句话同时给出了作品、字数、风格、搜索要求、角度选择、写作流程和改稿目标，比“帮我写读后感”稳定得多。

## 最推荐的分步使用方式

| 步骤 | 你要让 Agent 做什么 | 产物 |
|---:|---|---|
| 1 | 先搜索，不要写 | 高质量评论摘要 |
| 2 | 整理角度 | 3-5 个可写主判断 |
| 3 | 选主判断 | 一个最适合成文的中心观点 |
| 4 | 搭结构 | 文章骨架 |
| 5 | 写初稿 | 第一版读后感 |
| 6 | 降 AI 味 | 更自然的修订稿 |
| 7 | 最终检查 | 可提交版本 |

## 不推荐的使用方式

不推荐：

```text
帮我写《活着》读后感。
```

原因：

- 信息太少。
- 容易生成模板文。
- 容易出现剧情概括。
- 容易写成“苦难、人性、生命意义”这类空泛内容。
- 很难体现真实阅读体验。

更推荐：

```text
我读完《活着》后最难受的是福贵最后只剩下老牛作伴。请围绕“苦难不是把人变伟大，而是把人一点点磨空”这个角度，帮我写一篇 1000 字读后感。要求不要写成励志文，不要使用 AI 腔词。
```

## 推荐输入模板

```text
作品名称：
读完程度：
字数要求：
提交场景：
希望风格：
已有感受：
必须提到的人物/情节：
不想写的角度：
是否允许联网搜索：
是否需要降低 AI 味：
```

输入越具体，生成结果越不像模板文。尤其是下面三项很重要：

| 信息 | 作用 |
|---|---|
| 已有感受 | 保留你的声音 |
| 不想写的角度 | 避免俗套 |
| 提交场景 | 控制文章像课堂作业、知乎回答、豆瓣长评还是个人随笔 |

## 适合输入哪些信息

- 你读完以后最记得的人物或情节。
- 你觉得不舒服、感动、困惑或反感的地方。
- 老师或平台的字数要求。
- 文章要交给谁看。
- 你不想写成什么样。
- 你已有的一两句话观点。
- 你的草稿。

## 如果用户没有读完书该怎么办

不要假装读完。

推荐输入：

```text
我还没有完整读完《卡拉马佐夫兄弟》，只读到一半。请不要替我伪装完整读后感，先帮我建立理解框架：主要人物关系、核心冲突、后续阅读时应该注意的问题，以及哪些角度适合读完后再写。
```

Agent 应该做：

- 建立阅读框架。
- 标出后续需要注意的细节。
- 提醒哪些判断现在还不能下。
- 不写“我读完全书后”的虚假语气。

## 如果用户已经有草稿该怎么办

推荐输入：

```text
下面是我的读后感草稿。请不要重写成另一篇文章，而是在尽量保留我原意的基础上，帮我删掉 AI 腔、套话和空泛总结，让它更像一个大学生认真读完书后的自然表达。
```

Agent 应该做：

- 保留你的主判断和基本语气。
- 标出最像 AI 的句子。
- 直接改写套话。
- 不把文章改成完全陌生的风格。

## 如果用户只想降低 AI 味该怎么办

推荐输入：

```text
请只帮我降低这篇读后感的 AI 味，不要大改观点。重点处理：套话、过度整齐的句式、空泛升华、万能结尾和不像真人阅读体验的地方。
```

Agent 应该优先：

1. 删除“体现了、揭示了、发人深省、总而言之”。
2. 打散过于整齐的句子。
3. 加入少量真实阅读过程。
4. 用具体人物和情节替代抽象概念。
5. 保留你的原意。

## 如果用户想写得更像知乎、豆瓣或课堂作业

### 知乎长回答风格

```text
我想把《百年孤独》的读后感写成知乎长回答风格。请先联网搜索知乎上相关高赞回答的常见写法，学习它们的结构和节奏，但不要复制句子。然后帮我写一个有个人判断、有阅读体验的版本。
```

特点：

- 开头有明确判断。
- 喜欢纠正常见误读。
- 段落推进比较清楚。
- 允许适度口语。

### 豆瓣长评风格

```text
我想写一篇偏豆瓣长评风格的《局外人》读后感。希望更像私人阅读记录，但也要有清楚观点。请先学习豆瓣长评常见节奏，再帮我搭结构。
```

特点：

- 更重个人阅读感。
- 可以有犹豫和重读发现。
- 不必太像标准议论文。

### 课堂提交版

```text
我要交一篇课堂读后感，作品是《活着》，字数 1000 字。要求不能太散文，也不能太论文。请帮我写成适合大学课程提交的版本，语言自然，有观点，有文本细节，不要像 AI 代写。
```

特点：

- 结构清楚。
- 观点明确。
- 不过度口语。
- 有人物和情节支撑。
- 不使用太网感的表达。

## 如果用户想写得更像自己

先输入你的原话，不要急着让 Agent 写。

推荐输入：

```text
我想让这篇读后感更像我自己写的。下面是我读完后的零散感受，请先帮我整理，不要把语气改得太成熟，也不要写得像标准范文。
```

最好提供：

- 你原本会怎么说。
- 你不喜欢的表达。
- 你读到某一幕时的真实反应。
- 你希望文章保留的笨拙或犹豫。

## 场景一：刚读完一本书，还没有思路

```text
我刚读完《生死疲劳》，想写一篇 1200 字读后感，但现在只有一些零散感受：轮回、土地、苦难、荒诞、历史压迫感。请先不要直接写文章，先帮我整理这些感受，提炼 3 个可写主判断。
```

## 场景二：已有一个大概观点

```text
我想写《悲惨世界》，核心观点是：它真正打动人的不是苦难本身，而是在极端苦难中依然有人选择善。请帮我围绕这个观点搭结构，并指出这个角度容易写俗的地方。
```

## 场景三：已经有草稿，需要改得不像 AI

```text
下面是我的读后感草稿。请不要重写成另一篇文章，而是在尽量保留我原意的基础上，帮我删掉 AI 腔、套话和空泛总结，让它更像一个大学生认真读完书后的自然表达。
```

## 场景四：想写知乎风格

```text
我想把《百年孤独》的读后感写成知乎长回答风格。请先联网搜索知乎上相关高赞回答的常见写法，学习它们的结构和节奏，但不要复制句子。然后帮我写一个有个人判断、有阅读体验的版本。
```

## 场景五：想写课堂提交版

```text
我要交一篇课堂读后感，作品是《活着》，字数 1000 字。要求不能太散文，也不能太论文。请帮我写成适合大学课程提交的版本，语言自然，有观点，有文本细节，不要像 AI 代写。
```

## 最后提醒

读后感最重要的不是“写得高级”，而是让人相信你真的读过、真的停下来想过。这个 Skill 的作用，是把这种停顿整理成文章。
