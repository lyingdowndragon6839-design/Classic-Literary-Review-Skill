# Outline

结构不是把"主题、人物、情节、意义"分成几段，而是让一个主判断逐步变得可信。

## 主判断

先写清楚文章到底要证明什么：

```text
这篇读后感要写的不是【常见主题】，而是【更具体的矛盾或刺痛点】。
我会用【人物/情节/细节 1】、【人物/情节/细节 2】、【人物/情节/细节 3】来支撑。
最后回到【读完后改变的理解】。
```

## Outline 模板

| 段落 | 功能 | 证据 | 读者感受 |
|---:|---|---|---|
| 1 | 从刺痛点进入，不从作者生平进入 | | |
| 2 | 提出主判断 | | |
| 3 | 第一处证据 | | |
| 4 | 深化判断 | | |
| 5 | 加入阅读过程或反思转弯 | | |
| 6 | 回到场景，而不是喊口号 | | |

## 剧情预算

剧情是证据，不是正文主体。如果超过五分之一的篇幅都在解释发生了什么，
文章大概率已经丢掉了判断。

只在以下情况写剧情：

- 这个场景能证明主判断
- 这个动作改变了人物理解
- 这个细节让读者的判断变得更复杂

## 失败信号

- 每段都能独立成立，但段落之间没有推进
- 主判断换一本书名也能成立
- 结尾靠"启示、意义、精神、光辉"收束
- 最有力量的细节没有服务主判断
