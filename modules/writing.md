# Writing Module

初稿只在主判断和结构确定后开始。结构阶段见 [outline.md](outline.md)。

很多读后感写坏，是因为它们一开始就落进四个安全格子：主题、人物、情节、意义。
这样的结构看起来完整，却经常把文章写平。更好的初稿应该从一个压力点长出来。

## Drafting Principles

| Principle | What It Means |
|---|---|
| Start from pressure | 从刺痛、不适、疑问或反直觉处进入 |
| Keep plot functional | 剧情只用于证明判断 |
| Preserve user voice | 保留用户原始感受里的粗粝和停顿 |
| Avoid textbook categories | 不按“主题分析、人物分析、艺术特色”分段 |
| Close on a scene | 结尾回到具体场景，不喊口号 |

## Paragraph Jobs

| Paragraph | Job | Risk |
|---:|---|---|
| 1 | 给读者一个继续读的理由 | 从作者生平或名著评价开头 |
| 2 | 明确主判断 | 观点太大、太顺、无法论证 |
| 3 | 展开第一处证据 | 变成剧情复述 |
| 4 | 让判断更复杂 | 只重复上一段 |
| 5 | 放入阅读过程 | 伪装个人感受 |
| 6 | 收束到场景或问题 | 万能人生道理结尾 |

## Plot Budget

剧情是证据，不是主体。只在它回答这些问题时写剧情：

- 这个场景为什么重要？
- 这个动作暴露了什么？
- 这个细节怎样改变读者对人物或作品的判断？

如果剧情解释超过全文五分之一，先删，再把必要情节压成证据句。

## Drafting Checklist

- [ ] 第一段不是作者简介。
- [ ] 主判断可以被争辩，而不只是被赞美。
- [ ] 每个长段落都有具体场景、动作或物件。
- [ ] 至少一句话来自用户真实阅读感受。
- [ ] 结尾没有“启示我们”“值得思考”“升华主题”。
