# Writing

初稿只在主判断和结构确定后开始。结构阶段见 [outline.md](outline.md)。

## 起草原则

| 原则 | 含义 |
|---|---|
| 从压力点开始 | 从刺痛、不适、疑问或反直觉处进入 |
| 剧情服务于判断 | 剧情只用于证明判断 |
| 保留用户声音 | 保留用户原始感受里的粗粝和停顿 |
| 不套教科书格子 | 不按"主题分析、人物分析、艺术特色"分段 |
| 停在场景上 | 结尾回到具体场景，不喊口号 |

## 段落功能

| 段落 | 功能 | 风险 |
|---:|---|---|
| 1 | 给读者一个继续读的理由 | 从作者生平或名著评价开头 |
| 2 | 明确主判断 | 观点太大、太顺、无法论证 |
| 3 | 展开第一处证据 | 变成剧情复述 |
| 4 | 让判断更复杂 | 只重复上一段 |
| 5 | 放入阅读过程 | 伪装个人感受 |
| 6 | 收束到场景或问题 | 万能人生道理结尾 |

## 剧情预算

剧情只在它能回答以下问题时才写：

- 这个场景为什么重要？
- 这个动作暴露了什么？
- 这个细节怎样改变读者对人物或作品的判断？

如果剧情解释超过全文五分之一，先删，再把必要情节压成证据句。

## 起草检查

- [ ] 第一段不是作者简介
- [ ] 主判断可以被争辩，而不只是被赞美
- [ ] 每个长段落都有具体场景、动作或物件
- [ ] 至少一句话来自用户真实阅读感受
- [ ] 结尾没有"启示我们""值得思考""升华主题"
