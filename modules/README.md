# Modules

`modules/` 存放内部执行手册。顶层文件负责入口和概览，这里的文件负责具体步骤、
  模板和判断标准。

## 模块索引

| Module | 用途 |
|---|---|
| [system.md](system.md) | 角色边界、输出契约和语言红线 |
| [decision-tree.md](decision-tree.md) | 不同用户场景的处理路径 |
| [research.md](research.md) | 联网搜索、来源筛选和 Research Summary |
| [prompt-learning.md](prompt-learning.md) | 从公开 prompt 仓库学习流程设计 |
| [style-learning.md](style-learning.md) | 风格学习和 Style Bank |
| [outline.md](outline.md) | 从主判断到文章结构 |
| [writing.md](writing.md) | 初稿写作 |
| [revision.md](revision.md) | 结构、证据和声音修订 |
| [humanization.md](humanization.md) | 降低 AI 味的轮次改稿 |
| [ai-check.md](ai-check.md) | AI 痕迹、逻辑跳跃和空洞结尾检查 |
| [reviewer.md](reviewer.md) | 文学编辑式审稿 |
| [prompt-library.md](prompt-library.md) | 可复制的用户提示词 |

## 维护原则

模块文件保持独立、短小、可执行。需要示例时优先链接到 [../examples/](../examples/)，
  不要在多个模块里复制同一段长示例。
