# Contributing

感谢你愿意改进这个仓库。这个项目更看重可执行的写作流程、真实案例和文档质量，不需要增加大量泛化 Prompt。

## 可以贡献什么

- 修复 broken link、排版、错别字和 Markdown 渲染问题。
- 补充一部作品的完整示例。
- 改进研究、结构、审稿或降 AI 味流程。
- 增加更清晰的质量检查项。
- 提交维护脚本或文档健康检查建议。

## Issue 规范

提交 Issue 时请尽量说明：

| 字段 | 说明 |
|---|---|
| Problem | 你遇到的问题或想改进的地方 |
| File | 相关文件路径 |
| Expected | 你认为更合适的行为或文档表现 |
| Evidence | 截图、链接、复现步骤或示例输入 |

## Pull Request 规范

提交 PR 前请确认：

- [ ] 文件命名使用小写 kebab-case。
- [ ] Markdown 文件只有一个一级标题。
- [ ] 新增链接都能打开。
- [ ] 新增示例不复制外部文章。
- [ ] README、`skill.md` 或索引文件在必要时同步更新。
- [ ] 变更范围清楚，没有混入无关重写。

## Commit Message

推荐使用简洁的 Conventional Commits：

```text
docs: fix broken links in README
refactor: move writing modules into modules directory
feat: add ordinary-world example
chore: update markdown audit checklist
```

## 内容风格

请保持自然、克制、可执行。避免营销化表达，也避免把说明写成空泛宣言。每一段最好能回答一个具体问题：它能帮助读者怎样更好地写、改或审一篇读后感？
