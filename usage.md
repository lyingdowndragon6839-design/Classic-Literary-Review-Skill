# 使用指南

给真正要写读后感的人看。你不需要理解所有内部模块，只需要把阅读情况说清楚，
 让 Agent 先帮你整理，再进入写作。

## 推荐的一句话输入

```text
我想写《悲惨世界》的读后感，字数 1500 字，风格要自然，像一个认真读完书的人写的。
请先联网搜索高质量评论，帮我提炼 3 个可写角度，再选一个最有力量的主判断，
搭结构，写初稿，最后帮我降低 AI 味。
```

这句话同时给出作品、字数、风格、搜索要求、角度选择、写作流程和改稿目标。

## 推荐的分步方式

| 步骤 | 让 Agent 做什么 | 产物 |
|---:|---|---|
| 1 | 先搜索，不要写 | 高质量评论摘要 |
| 2 | 整理角度 | 3-5 个可写主判断 |
| 3 | 选主判断 | 一个最适合成文的中心观点 |
| 4 | 搭结构 | 文章骨架 |
| 5 | 写初稿 | 第一版读后感 |
| 6 | 降 AI 味 | 更自然的修订稿 |
| 7 | 最终检查 | 可提交版本 |

## 不推荐的输入

```text
帮我写《活着》读后感。
```

信息太少，容易生成模板文、剧情概括和"苦难、人性、生命意义"式空泛内容。

更推荐：

```text
我读完《活着》后最难受的是福贵最后只剩下老牛作伴。
请围绕"苦难不是把人变伟大，而是把人一点点磨空"这个角度，
帮我写一篇 1000 字读后感。
要求不要写成励志文，不要使用 AI 腔词。
```

## 输入模板

```text
作品名称：
读完程度：
字数要求：
提交场景：
希望风格：
已有感受：
必须提到的人物/情节：
不想写的角度：
是否允许联网搜索：
是否需要降低 AI 味：
```

| 信息 | 作用 |
|---|---|
| 已有感受 | 保留你的声音 |
| 不想写的角度 | 避免俗套 |
| 提交场景 | 控制文章像课堂作业、知乎回答、豆瓣长评还是个人随笔 |

## 常见场景

### 没有读完书

不要假装读完。可以这样问：

```text
我还没有完整读完《卡拉马佐夫兄弟》，只读到一半。
请不要替我伪装完整读后感，先帮我建立理解框架：
主要人物关系、核心冲突、后续阅读时应该注意的问题，
以及哪些角度适合读完后再写。
```

Agent 应该建立阅读框架，提醒哪些判断现在还不能下，不写"我读完全书后"的虚假语气。

### 已经有草稿

```text
下面是我的读后感草稿。请不要重写成另一篇文章，而是在尽量保留我原意的基础上，
帮我删掉 AI 腔、套话和空泛总结，让它更像一个大学生认真读完书后的自然表达。
```

Agent 应该保留主判断和基本语气，标出最像 AI 的句子，直接改写套话。

### 只想降低 AI 味

```text
请只帮我降低这篇读后感的 AI 味，不要大改观点。
重点处理：套话、过度整齐的句式、空泛升华、万能结尾
和不像真人阅读体验的地方。
```

优先处理：

1. 删除"体现了、揭示了、发人深省、总而言之"
2. 打散过于整齐的句子
3. 加入少量真实阅读过程
4. 用具体人物和情节替代抽象概念
5. 保留原意

## 场景风格

| 场景 | 写法边界 |
|---|---|
| 课堂提交 | 结构清楚，观点明确，不过度口语 |
| 知乎长回答 | 开头锋利，适合纠正常见误读 |
| 豆瓣长评 | 更重个人阅读感，可以保留犹豫和重读发现 |
| 公众号文章 | 节奏更完整，但不要写成励志文 |
| 演讲稿 | 保留一个主判断，减少复杂论证，增加停顿 |

## 最后提醒

读后感最重要的不是"写得高级"，而是让人相信你真的读过、真的停下来想过。
