# Quality Check

写作质量检查清单。交付一篇读后感之前逐项确认。

仓库维护者的检查清单见 [docs/maintenance.md](docs/maintenance.md)。

## 写作质量

- [ ] 文章有一个明确主判断
- [ ] 剧情复述不超过全文 20%
- [ ] 主要论点都有具体人物、情节、物件或叙述方式支撑
- [ ] 用户自己的阅读感受没有被外部评论盖住
- [ ] 文章没有用"人性、命运、意义"填补论证空白
- [ ] 结尾回到具体场景、人物或问题，而不是万能口号

## Humanization

- [ ] 已删除"首先、其次、最后、总而言之、由此可见"等模板连接词
- [ ] 段落长度和句子节奏有变化
- [ ] 至少有一个具体阅读过程或重读发现
- [ ] 抽象词被场景、动作、物件替代
- [ ] 修改没有抹掉用户原意

## AI Trace

- [ ] "受益匪浅、发人深省、体现了、揭示了"等套话已改写
- [ ] 没有空泛赞美
- [ ] 没有机械过渡
- [ ] 没有为深刻而深刻的句子
- [ ] 文章不能只替换书名就套到另一部作品

## Research Integrity

- [ ] 联网时有 Research Summary
- [ ] 无法访问的来源标记为 blocked
- [ ] 引用外部资料时区分观点和写法
- [ ] 没有复制、改写或拼接外部文章
- [ ] 离线模式没有声称"检索发现"

## 最终检查

- [ ] 读完一遍，找不到一句明显像模板的话
- [ ] 换一本书名，文章的主要论证不能直接成立
- [ ] 用户如果读到，会认出自己的感受
